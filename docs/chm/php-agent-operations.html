PHP Agent (HTTP) - Complete Guide

PHP Agent enables file operations over HTTP when SSH access is unavailable. Ideal for shared hosting environments.

Overview

What is PHP Agent?

  • Single PHP file (sftp.php) deployed to web server
  • Provides file operations via HTTP API
  • Secure HMAC-SHA256 authentication
  • Supports file listing, upload, download, delete, rename, mkdir
  • No SSH account required - works with web hosting

Capabilities

  • LIST: Directory listing with file metadata
  • STAT: File/directory information
  • GET: Download files with range support
  • PUT: Upload files with chunked transfer
  • DELETE: Remove files
  • MKDIR: Create directories (recursive)
  • RMDIR: Remove directories
  • RENAME: Rename/move files
  • HASH: Compute file hashes (SHA256, etc.)
  • SHELL_EXEC: Execute shell commands (if enabled)

Deployment Guide

Step 1: Configure Plugin

  1. Open connection dialog
  2. Select transfer mode: PHP Agent (HTTP)
  3. Enter URL: https://example.com/sftp.php
  4. Enter username (optional, for identification)
  5. Enter password (will be used as PSK - Pre-Shared Key)
  6. Save session

Step 2: Generate sftp.php

  1. After saving password, plugin generates sftp.php
  2. Located in plugin runtime directory (next to plugin DLL): ...\\plugins\\wfx\\sftpplug\\sftp.php (build also copies it to build\\bin\\sftp.php)
  3. Contains derived authentication constants
  4. Ready for upload to server

Step 3: Upload to Server

  1. Use FTP, control panel, or other method
  2. Upload sftp.php to web-accessible location
  3. Note the full URL for plugin configuration
  4. Ensure file permissions allow execution (755 typical)

Step 4: Test Connection

  1. Enter URL in plugin: https://example.com/sftp.php
  2. Click Connect
  3. Plugin probes endpoint for capabilities
  4. On success, ready for file operations

Authentication

PSK (Pre-Shared Key)

Three configuration methods (in order of preference):

  1. Environment Variable (Recommended): 
    AGENT_PSK=your_secret_key_here
    Environment keys checked: TC_SFTP_AGENT_PSK, SFTP_AGENT_PSK, AGENT_PSK
  2. Constant in sftp.php: 
    const AGENT_PSK = 'your_secret_key_here';
  3. Salted Hash (for deployment): 
    const AGENT_PSK_SALT = 'random_salt';
const AGENT_PSK_SHA256 = 'sha256(salt:psk)';
    Allows upload-only deployment without knowing plain PSK

HMAC Authentication

Each request includes authentication headers:

X-SFTP-Auth: legacy_token (optional)
X-SFTP-TS: timestamp
X-SFTP-Nonce: random_nonce
X-SFTP-Signature: HMAC-SHA256(base, psk)

Base string format:

METHOD\nOPERATION\nPATH\nTIMESTAMP\nNONCE

Security Features

  • Timestamp validation: Requests expire after 300 seconds
  • Nonce replay protection: Each nonce used once
  • HMAC-SHA256: Cryptographic signature
  • Path traversal protection: Root jail enforced

URL Format

Basic URL

https://example.com/sftp.php
https://example.com/path/to/sftp.php
https://example.com/sftp.php?path=/subdir

With Port

https://example.com:8443/sftp.php

Path Parameter

Can specify path in URL or via parameter:

https://example.com/sftp.php?path=files
https://example.com/sftp.php?path=%2Fvar%2Fwww

Operations Reference

PROBE - Capability Detection

Automatic on first connection:

GET https://example.com/sftp.php?op=PROBE

Response:
{
  "ok": true,
  "agent": { "name": "TC PHP Agent", "version": "1.0.1" },
  "runtime": { "php_version": "8.2", "sapi": "apache2handler" },
  "ini": { "upload_max_filesize": "256M", ... },
  "capabilities": { "upload": true, "download": true, ... }
}

Upload Chunk Policy

  • Auto mode: uses capabilities.recommended_chunk_size from PROBE
  • Safety clamp: plugin limits Auto to 1..64 MB
  • Fallback: if recommendation is unavailable, Auto uses 1 MB
  • Manual: 2 / 4 / 8 / 16 / 32 / 64 MB

LIST - Directory Listing

GET https://example.com/sftp.php?op=LIST&path=/subdir

Response (JSON):
{
  "ok": true,
  "path": "subdir",
  "items": [
    { "name": "file.txt", "is_dir": false, "size": 1234, "mtime": 1234567890, "perms": "0644" }
  ]
}

Response (Plain):
F\t1234\t1234567890\tbase64(filename)
D\t0\t1234567890\tbase64(dirname)

GET - Download File

GET https://example.com/sftp.php?op=GET&path=file.txt&offset=0&length=1024

Headers:
Content-Type: application/octet-stream
Content-Length: 1024
X-SFTP-File-Size: 5000
X-SFTP-Offset: 0
Accept-Ranges: bytes

PUT - Upload File

POST https://example.com/sftp.php?op=PUT&path=file.txt&offset=0

Headers:
Content-Type: multipart/form-data or raw POST data
X-SFTP-Content-SHA256: sha256hash (optional verification)

Response:
{
  "ok": true,
  "path": "file.txt",
  "written": 1024,
  "size": 5000,
  "sha256": "abc123...",
  "part": true
}

FINALIZE - Complete Upload

POST https://example.com/sftp.php?op=FINALIZE&path=file.txt&overwrite=1

Response:
{
  "ok": true,
  "path": "file.txt",
  "size": 5000,
  "finalized": true
}

DELETE - Remove File

POST https://example.com/sftp.php?op=DELETE&path=file.txt

Response:
{ "ok": true, "deleted": true, "path": "file.txt" }

MKDIR - Create Directory

POST https://example.com/sftp.php?op=MKDIR&path=newdir&recursive=1&mode=0775

Response:
{ "ok": true, "created": true, "path": "newdir" }

RMDIR - Remove Directory

POST https://example.com/sftp.php?op=RMDIR&path=olddir&recursive=0

Response:
{ "ok": true, "removed": true, "path": "olddir", "recursive": false }

RENAME - Move/Rename

POST https://example.com/sftp.php?op=RENAME&from=old.txt&to=new.txt&overwrite=1

Response:
{ "ok": true, "renamed": true, "from": "old.txt", "to": "new.txt" }

HASH - Compute File Hash

GET https://example.com/sftp.php?op=HASH&path=file.txt&algo=sha256

Response:
{ "ok": true, "path": "file.txt", "algo": "sha256", "hash": "abc123..." }

SHELL_EXEC - Execute Command

POST https://example.com/sftp.php?op=SHELL_EXEC&cmd=ls+-la&cwd=/var/www

Response:
{
  "ok": true,
  "cwd": ".",
  "cwd_abs": "/var/www",
  "method": "shell_exec",
  "exit_code": 0,
  "stdout_b64": "base64_encoded_output",
  "stderr_b64": "base64_encoded_errors"
}

PHP Shell Console

Overview

Pseudo-terminal interface for command execution:

  • HTTP-based command execution
  • Working directory tracking
  • Command history (Up/Down arrows)
  • Output colorization
  • Basic completion suggestions

Opening Shell

  1. Connect with PHP Agent or PHP Shell mode
  2. Click Shell... button
  3. Plugin validates authentication first
  4. Console opens with prompt showing host and path

Shell Features

  • Command history: Up/Down arrows
  • Tab completion: Basic filename completion
  • Working directory: Displayed in prompt
  • Output separation: Command vs output coloring

Limitations

  • Not a full PTY/TTY emulator
  • Interactive programs may not work (vi, top)
  • Some terminal escape sequences not handled

Configuration Options

PHP Server Requirements

  • PHP 7.4 or later (PHP 8.x recommended)
  • File functions enabled
  • Directory functions enabled
  • Hash extension available
  • JSON extension available

PHP INI Limits

  • upload_max_filesize: Maximum single upload
  • post_max_size: Maximum POST data
  • max_execution_time: Script timeout
  • memory_limit: Memory per request

Agent Constants

AGENT_NAME = 'TC PHP Agent'
AGENT_VERSION = '1.0.0'
AGENT_PSK = 'CHANGE_ME_TO_LONG_RANDOM_SECRET'
AGENT_PSK_SALT = ''
AGENT_PSK_SHA256 = ''
AGENT_ROOT = __DIR__  (or custom path)
AGENT_NONCE_TTL = 300
AGENT_DEFAULT_CHUNK = 262144
AGENT_MAX_LIST_ITEMS = 20000

Troubleshooting

HTTP 401 - Authentication Failed

  • Verify password matches PSK in sftp.php
  • Check timestamp synchronization
  • Ensure HMAC signature computed correctly

HTTP 403 - Path Escape Attempt

  • Requested path goes outside AGENT_ROOT
  • Adjust AGENT_ROOT if needed
  • Use relative paths within allowed root

HTTP 404 - Not Found

  • Verify URL points to actual sftp.php location
  • Check file permissions (executable)
  • Ensure mod_rewrite not interfering

HTTP 503 - Service Unavailable

  • AGENT_PSK not configured
  • Set environment variable or constant
  • Deploy properly generated sftp.php

Upload Fails

  • Check PHP upload limits (upload_max_filesize)
  • Verify disk space on server
  • Check directory permissions
  • Review PHP error logs

Download Returns Empty File

  • Verify file is readable
  • Check open_basedir restrictions
  • Ensure file exists at specified path
Security Warning: Always use HTTPS for PHP Agent in production. Plain HTTP exposes all traffic including authentication.
Tip: Use environment variables for PSK in production. Never commit sftp.php with real secrets to version control.
