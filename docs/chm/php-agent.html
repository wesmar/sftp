PHP Agent (HTTP)

PHP Agent allows transfer operations without SSH account access. Ideal for shared hosting environments where only web access is available.

How it works

  • Single PHP file (sftp.php) deployed to web server
  • Plugin communicates via HTTP with HMAC-SHA256 authentication
  • Supports full file operations: list, upload, download, delete, rename, mkdir
  • No SSH daemon needed - works with any PHP-enabled web server

Quick start

  1. In plugin, select transfer mode: PHP Agent (HTTP)
  2. Enter URL: https://example.com/sftp.php
  3. Enter username and password
  4. Save session - plugin generates sftp.php with your auth settings
  5. Upload sftp.php to your web server
  6. Connect - plugin probes endpoint and verifies capabilities

Deployment flow

  1. Choose transfer mode PHP Agent (HTTP) (or PHP Shell (HTTP))
  2. Enter password in plugin options and save session
  3. Plugin updates local sftp.php with generated auth material
  4. Upload local sftp.php to server path used in connection URL
  5. Test connection from plugin

URL format

https://example.com/sftp.php
https://example.com/path/to/sftp.php
https://example.com:8443/sftp.php

Authentication

Client side (Total Commander)

  • Password stored encrypted with DPAPI in sftpplug.ini
  • Used to compute HMAC-SHA256 signatures for each request
  • Timestamp and nonce prevent replay attacks

Server side (sftp.php)

  • PSK (Pre-Shared Key) configured via environment variable or constant
  • Optional: Salted hash for deployment without exposing plain PSK
  • HMAC verification on every request

Probe endpoint

The plugin probes endpoint capabilities before full use:

  • PHP version detection
  • INI limits (upload_max_filesize, memory_limit, etc.)
  • Recommended upload chunk size (capabilities.recommended_chunk_size)
  • Available operations (upload, download, shell_exec, etc.)
  • Filesystem permissions

Capabilities

  • LIST: Directory listing with metadata (name, size, mtime, permissions)
  • STAT: File/directory information
  • GET: Download with range support (resume)
  • PUT: Upload with chunked transfer
  • DELETE: Remove files
  • MKDIR: Create directories (recursive)
  • RMDIR: Remove directories
  • RENAME: Rename/move files
  • HASH: Compute file hashes (SHA256, etc.)
  • SHELL_EXEC: Execute shell commands (if enabled on server)

PHP Shell Console

For servers with shell_exec enabled:

  • HTTP pseudo-terminal for command execution
  • Working directory tracking
  • Command history (Up/Down)
  • Basic completion suggestions
  • Colored output separation

Access via Shell... button after connecting.

Security considerations

  • Always use HTTPS in production - plain HTTP exposes all traffic
  • PSK stored as salted hash option for deployment
  • Path traversal protection (root jail)
  • Timestamp validation (300 second window)
  • Nonce replay protection

Server requirements

  • PHP 7.4 or later (PHP 8.x recommended)
  • File functions enabled (fopen, fread, fwrite, etc.)
  • Directory functions enabled (opendir, readdir, etc.)
  • Hash extension available
  • JSON extension available
  • For shell exec: shell_exec/proc_open not in disable_functions

PHP INI limits

Be aware of server PHP configuration:

  • upload_max_filesize - Maximum single upload size
  • post_max_size - Maximum POST data size
  • max_execution_time - Script timeout
  • memory_limit - Memory per request
  • open_basedir - Directory restrictions

Chunk Selection (PHP Agent Upload)

  • Auto: Plugin reads recommended_chunk_size from PROBE and applies safety clamp 1..64 MB
  • Auto fallback: When recommendation is missing/unavailable, plugin uses 1 MB
  • Manual options: 2 / 4 / 8 / 16 / 32 / 64 MB

Troubleshooting

  • 401 Unauthorized: Password doesn't match PSK, check sftp.php deployment
  • 404 Not Found: URL incorrect or sftp.php not at that path
  • 503 Service Unavailable: PSK not configured in sftp.php
  • 500 Internal Error: Check PHP error logs, file permissions
  • Upload fails: Check upload_max_filesize, post_max_size, disk space
