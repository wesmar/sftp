Proxy Configuration

The plugin supports various proxy types for SSH connections. Configure proxy settings via the Proxy... button in the main connection dialog.

Supported Proxy Types

No Proxy

Direct connection to the target server. Default setting.

HTTP CONNECT Proxy

  • Uses HTTP CONNECT method to tunnel SSH through HTTP proxy
  • Supports authentication (Basic)
  • Example: proxy.company.com:8080
  • Username/password stored encrypted

SOCKS4a Proxy

  • SOCKS version 4a protocol
  • Supports domain name resolution (the "a" suffix)
  • No authentication support
  • Example: socks4a://proxy:1080

SOCKS5 Proxy

  • Full SOCKS version 5 protocol
  • Supports authentication (username/password)
  • Supports IPv4, IPv6, and domain names
  • Example: socks5://user:pass@proxy:1080

System Default Proxy

Uses Windows system proxy settings from Internet Options.

Proxy Configuration Fields

  • Proxy server:
    • Format: hostname or hostname:port
    • Default ports: 8080 (HTTP), 1080 (SOCKS)
  • Username: Optional proxy authentication username
  • Password: Optional proxy authentication password (encrypted storage)

Proxy Authentication

When proxy requires authentication:

  1. Enter username in Username field
  2. Enter password in Password field
  3. Password is stored encrypted via DPAPI
  4. For SOCKS5, authentication is negotiated during connection

Proxy with SSH Key Authentication

Proxy settings are independent of SSH authentication. You can use:

  • Proxy + password authentication
  • Proxy + public key authentication
  • Proxy + Pageant (SSH agent)

Troubleshooting

Proxy connection fails immediately

  • Verify proxy server is reachable: telnet proxy:port
  • Check proxy type matches server capability
  • Verify credentials if authentication required

Connection works without proxy but not with

  • Check proxy server logs for denied connections
  • Verify proxy allows CONNECT to target port (usually 22)
  • Try SOCKS5 if HTTP CONNECT fails

SOCKS5 authentication fails

  • Ensure SOCKS5 (not SOCKS4a) is selected
  • Verify username/password are correct
  • Check if proxy requires specific authentication method
Security note: Proxy passwords are stored encrypted, but the proxy connection itself may not be encrypted. Use SOCKS5 or HTTPS proxy for sensitive environments.
