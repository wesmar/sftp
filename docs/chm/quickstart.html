Quick Start

  1. Open connection dialog and enter host/URL.
  2. Set username and authentication (password or key).
  3. Choose transfer mode based on host capabilities.
  4. Save named session and connect.

Three common scenarios

A) Normal SSH server

  • Transfer mode: SSH default.
  • Optional: Pageant or PEM/PPK key.

B) Restricted SSH host (SFTP/SCP issues)

  • Start with SCP-for-data, then SCP-only if needed.
  • Enable shell transfer fallback for stubborn hosts.

C) Hosting without SSH account

  • Use PHP Agent (HTTP).
  • Upload generated sftp.php and connect by URL.

D) SSH server only reachable through bastion

  • Keep transfer mode as SSH (SFTP/SCP).
  • Enable Use jump host and configure the bastion in Jump....
  • Authenticate to jump host and target host independently.

E) Two Windows machines on the same LAN

  • Use LAN Pair (SMB-like).
  • Choose Auto, Receiver, or Donor.
  • Click Pair... and confirm the role when the peer is found.
