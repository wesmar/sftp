Transfer Modes Decision Guide

1) SSH (SFTP/SCP)

Use first when regular SSH access works. Best default for most servers.

2) Use SCP for data transfers

Use when listing via SFTP works, but upload/download over SFTP is unstable.

3) Use SCP for everything (no SFTP)

Use when SFTP subsystem is blocked or broken, but SCP command path is available.

4) Use shell transfer (for blocked hosts)

Use when even SCP is problematic. Plugin can switch to shell stream/chunk strategies.

5) PHP Agent (HTTP)

Use when no shell/SSH account is available. Requires uploaded sftp.php.

6) PHP Shell (HTTP)

Use for command execution and maintenance through HTTP pseudo-terminal.

7) LAN Pair (SMB-like)

Use for direct Windows-to-Windows pairing on the local network. This mode does not use SSH, PHP, or the system SMB stack.

Jump Host note

Jump Host / ProxyJump is not a separate transfer mode. It is an SSH transport extension for the standard SSH (SFTP/SCP) mode when the target host is reachable only through a bastion server.

Fast practical selection

  1. Start with SSH default.
  2. If transfer fails: enable SCP for data.
  3. If still failing: SCP only.
  4. If host blocks channels: shell transfer fallback.
  5. If SSH unavailable: PHP Agent/PHP Shell.
  6. If target SSH is behind a bastion: keep SSH mode and enable Jump Host.
  7. If you need local Windows-to-Windows pairing: use LAN Pair.
