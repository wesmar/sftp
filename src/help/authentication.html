Authentication Methods

The plugin supports multiple SSH authentication methods for maximum compatibility with different server configurations.

Available Authentication Methods

Password Authentication

  • Standard username/password login
  • Password stored encrypted via DPAPI
  • Supports servers with keyboard-interactive
  • Most widely supported method

Public Key Authentication

  • Uses SSH key pair (public/private)
  • More secure than password
  • Supports multiple key formats
  • Can be combined with passphrase

SSH Agent (Pageant)

  • Uses keys loaded in Pageant
  • No need to specify key files
  • Non-intrusive: tries automatically
  • Falls back to other methods if needed

Keyboard Interactive

  • Challenge-response authentication
  • Used by some hosting providers
  • May involve multiple prompts
  • Often used for 2FA

Key File Formats

PEM Format (OpenSSH)

  • Standard OpenSSH format
  • Generated by ssh-keygen
  • File extension: .pem or no extension
  • Both RSA and ECDSA supported

PPK Format (PuTTY)

  • PuTTY native format
  • Generated by PuTTYgen
  • File extension: .ppk
  • Native conversion: Plugin converts PPK internally, no puttygen.exe needed
  • Supports PPK v2 and v3
  • PPK v3 uses Argon2 key derivation (requires argon2.dll)

Key Format Detection

The plugin automatically detects key format:

  • Reads key file header
  • Handles both PEM and PPK seamlessly
  • Converts to internal format for libssh2

Using Public Key Authentication

Setup Steps

  1. Generate key pair (OpenSSH or PuTTYgen)
  2. Copy public key to server: ~/.ssh/authorized_keys
  3. In plugin, browse to private key file
  4. If key has passphrase, enter when prompted
  5. Optional: Save session with key path

Key File Selection

  • Click Private key file button
  • Browse to key location
  • Both .pem and .ppk visible in file dialog
  • Public key file optional (auto-detected if same name)

Passphrase Handling

  • Passphrase prompted when connecting
  • Not stored (for security)
  • Can use Pageant to cache passphrase

Using Pageant (SSH Agent)

Setup

  1. Start Pageant (comes with PuTTY)
  2. Load your private key(s) into Pageant
  3. In plugin, check Use SSH agent (Pageant)
  4. Connect - plugin automatically tries Pageant keys

How It Works

  • Plugin queries Pageant for available keys
  • Server lists accepted authentication methods
  • Plugin tries each key from Pageant
  • Logs show which keys were attempted
  • Falls back to password if no key accepted

Pageant Log Messages

Agent: Found 2 key(s) in Pageant
Agent: Trying key "rsa-key-2024"
Agent: Authentication succeeded with key "rsa-key-2024"

Authentication Flow

Server Capability Detection

On connection, server advertises supported methods:

  • publickey - Key authentication available
  • password - Password authentication available
  • keyboard-interactive - Challenge-response available

Plugin Authentication Order

  1. Pageant keys (if enabled)
  2. Specified private key file
  3. Password authentication
  4. Keyboard interactive (if required)

Combined Authentication

Some servers require multiple authentication steps:

  • Key + password (uncommon)
  • Key + 2FA code (via keyboard-interactive)
  • Plugin handles multi-stage authentication

Common Issues

Key authentication fails

  • Verify public key is in ~/.ssh/authorized_keys
  • Check key permissions on server (600 or 644)
  • Ensure correct key format (PEM/PPK)
  • Try with Pageant to isolate issue

PPK conversion fails

  • Ensure PPK file is not corrupted
  • For PPK v3, ensure argon2.dll is available
  • Try converting with puttygen as workaround

Pageant not detected

  • Ensure Pageant is running
  • Check key is loaded in Pageant
  • Verify "Use SSH agent" is checked

Keyboard interactive prompts

  • Enter responses as prompted
  • May be multiple prompts in sequence
  • Common for 2FA or hosting panels

Security Recommendations

  • Use key authentication when possible
  • Protect private keys with passphrase
  • Use Pageant to cache unlocked keys
  • Don't store passphrases in sessions
  • Regular key rotation for sensitive systems
PPK v3 Support: PPK version 3 uses Argon2 key derivation. Ensure argon2.dll is in the plugin directory for PPK v3 support.
