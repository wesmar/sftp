Import from PuTTY / WinSCP

The Import... button in the connection dialog opens a wizard that migrates sessions from PuTTY or WinSCP into the plugin without modifying the source data.

Supported sources

SourceLocation
PuTTY (registry)HKCU\Software\SimonTatham\PuTTY\Sessions
PuTTY Portable (folder)Browse to the PuTTY Portable folder — the plugin finds putty.reg automatically (recursive search up to 4 levels deep)
WinSCP (registry)HKCU\Software\Martin Prikryl\WinSCP 2\Sessions
WinSCP Portable (.ini)Browse to WinSCP.ini directly

Checkbox picker dialog

After scanning the source, the plugin shows a checkbox list of all found sessions:

  • Every session has an individual checkbox — pick any combination.
  • Select All and Deselect All buttons for bulk operations.
  • Only checked sessions are imported; the rest are skipped silently.
  • Already-existing sessions with the same name are not overwritten unless you explicitly re-check them.

Path memory (4-path history)

The folder/file browse dialog for portable sources remembers the last 4 used paths. They are persisted in the [ImportPaths] section of sftpplug.ini and pre-selected on next open, so you do not need to navigate to the same folder twice.

What is imported

  • Hostname and port
  • Username
  • Key file path (if set)
  • SSH agent flag

Passwords are not imported (PuTTY and WinSCP store them in formats incompatible with the plugin's DPAPI storage). You will be prompted for a password on the first connection.

Protocol mapping

  • WinSCP SFTP sessions → SSH (SFTP/SCP) mode.
  • WinSCP SCP sessions → scponly=1 and scpfordata=1.
  • Unsupported WinSCP protocols (FTP, WebDAV, S3) → skipped, reported in summary.

Key file compatibility

  • Imported key paths may point to .pem (OpenSSH) or .ppk (PuTTY) files.
  • The plugin converts PPK v2 and PPK v3 (Argon2) natively — no external puttygen needed.
  • Verify the path is still valid after import, especially for portable setups.

After import checklist

  1. Open an imported session with Alt+Enter.
  2. Verify transfer mode matches server capabilities.
  3. Check proxy settings if the host is behind a proxy.
  4. Enter password and save if needed.
  5. Test connection and adjust SCP/Shell options for restrictive hosts.
Tip: For PuTTY Portable, navigate to the folder that contains Data\settings\ or the root of the portable installation. The plugin recurses up to 4 directory levels looking for putty.reg.
