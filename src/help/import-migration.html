Import from PuTTY / WinSCP

What import reads

  • Registry sessions from PuTTY and WinSCP in current user profile.
  • Host, port, username, selected key path, and selected SSH-agent usage flags.

Protocol mapping

  • WinSCP SFTP sessions import as normal SSH mode.
  • WinSCP SCP sessions import with scponly=1 and scpfordata=1.
  • Unsupported WinSCP protocols are skipped and reported in summary.

Key compatibility

  • Imported key paths may include .pem or .ppk.
  • Plugin supports native PPK conversion for authentication workflow.

After import checklist

  1. Open imported session with Alt+Enter.
  2. Verify transfer mode and proxy.
  3. Test connection and adjust SCP/Shell options if host is restrictive.
