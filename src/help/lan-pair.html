LAN Pair

LAN Pair is a Windows-to-Windows transfer mode for direct local-network pairing without relying on the system SMB stack. It is designed for practical file exchange between trusted machines on the same network.

What LAN Pair is for

  • Quick transfer between two Windows machines on the same LAN
  • Cases where classic SMB pairing is inconvenient or blocked by local policy
  • Direct plugin-managed sessions inside Total Commander

What it is not

  • It is not classic Windows file sharing
  • It does not enumerate normal SMB shares
  • It is not routed through SSH, SFTP, PHP Agent, or PHP Shell

Discovery and network ports

LAN Pair uses two ports:

PortProtocolPurpose
45845UDP (broadcast)Peer discovery — both machines announce themselves on the local network every 2 seconds
45846TCPPairing handshake and file transfer session

Both ports must be open in Windows Firewall on both machines. If peers do not appear in the discovery list, this is the first thing to check.

Firewall tip: UDP broadcast (port 45845) is blocked between network segments and VLANs by default. LAN Pair discovery works reliably only within the same broadcast domain (same subnet/switch). If your machines are on different VLANs, peers will not be found via auto-discovery.

Trust model — first connection vs. subsequent connections

LAN Pair uses a TOFU (Trust On First Use) authentication model. You only need to enter a password once:

  1. First connection: A pairing password is required. Both sides must use the same password.
  2. On success, the plugin derives a cryptographic trust key (PBKDF2-HMAC-SHA256, 120,000 iterations) and stores it encrypted on disk via Windows DPAPI — bound to your Windows user account.
  3. Subsequent connections: The stored trust key is used automatically. No password prompt appears.

Trust is stored per peer pair. Removing a session or switching Windows accounts requires re-pairing with a password.

Note: The pairing password can be left empty if both machines agree on an empty password. In that case, trust is still established cryptographically — but an empty password offers no brute-force resistance. Use a real password on any network you do not fully control.

Roles

  • Auto — guided 60-second pairing wizard; the plugin starts the local file server in the background and waits up to 60 seconds for a peer to appear, then prompts you to connect or stay as server.
  • Donor — runs the local LAN Pair file server and waits for an incoming Receiver connection. Does not initiate outgoing connections — the other machine (configured as Receiver) must connect first. Use this on the machine whose files you want to access remotely.
  • Receiver — connects to the Donor's file server. Does not start a local file server. Use this on the machine that will browse and transfer files from the Donor.
Which role for which machine? The Donor is the machine that has the files — it runs as server and waits. The Receiver is the machine that connects and browses — it acts as client. Both machines must be running Total Commander with the plugin installed.

Auto mode — pairing wizard

When Auto is selected and you click Pair..., the plugin runs a guided wizard:

  1. The local file server starts listening on TCP 45846 in the background.
  2. A 60-second countdown dialog appears: "Searching for LAN Pair peer on local network...". You can cancel at any time.
  3. As soon as the other machine is detected, the dialog closes and a prompt appears:
    • Yes — connect to the discovered peer immediately (this machine acts as client / Receiver).
    • No — stay in server / Donor mode; the local file server keeps running.
    • Cancel — stop the local server and cancel.
  4. If no peer appears within 60 seconds, the local server is stopped automatically and a status message is shown. No ports are left open.
Tip: Both machines should click Pair... at roughly the same time. With a 60-second window and a 2-second broadcast interval, there is no need to rush.

Basic workflow

  1. Create or edit a session and set Transfer = LAN Pair.
  2. Set the session name. This name is also shown as the visible peer label on the other machine.
  3. Choose Auto, Receiver, or Donor in the role dropdown.
  4. Optional: set Session timeout in minutes. 0 means no limit.
  5. Click Pair....
  6. On first use: enter the pairing password (must match on both sides).
  7. When the peer is found and roles are confirmed, the session opens as a regular TC file panel.

TrustedInstaller — accessing protected system files

Windows protects certain system files and directories by assigning them to the TrustedInstaller service account (e.g. files in C:\Windows\, C:\Program Files\WindowsApps\, Windows Update caches). Even an Administrator account cannot modify these files directly without taking ownership. The Trusted Installer checkbox in the LAN Pair connection settings lets you work around this restriction for incoming connections.

How it works

  1. When a remote peer connects to the local LAN Pair file server, the server thread impersonates the TrustedInstaller service account before handling any commands.
  2. While impersonating TrustedInstaller, all file operations (read, write, delete, rename, mkdir) run with TrustedInstaller privileges, bypassing the normal ownership restrictions.
  3. Impersonation is reverted automatically when the remote peer disconnects.

The Trusted Installer checkbox has two independent effects depending on which role this machine plays:

  • As Donor (file server): incoming connections from a Receiver are served under TI impersonation — the Receiver can read, write, delete, and rename TI-owned files on this machine.
  • As Receiver (file client): outgoing file transfers run under TI impersonation on this machine — you can download files to TI-protected local directories and upload files from TI-protected local paths.

To work with TI-protected files on the remote machine (the Donor), the checkbox must be enabled on the Donor's connection profile, not the Receiver's.

Requirements

  • The process hosting the plugin (Total Commander) must be running with SeDebugPrivilege. TC normally has this privilege when run as Administrator or with the standard elevated token.
  • The TrustedInstaller Windows service must be startable. The plugin starts it automatically if it is not already running.
  • The setting takes effect immediately on the next incoming connection — no TC restart needed after enabling or disabling the checkbox.
Security note: TrustedInstaller impersonation grants write access to core Windows system files. Enable this option only on machines you fully control and only while you actually need it.

Session timeout

  • 0 = no timeout
  • >0 = session timeout in minutes, measured from the last operation
  • The timeout is stored per session as lanpairtimeout in sftpplug.ini
  • Timeout is checked before each operation; active file transfers are not interrupted mid-stream

UI behavior

  • In LAN Pair mode, SSH-specific controls are disabled (key files, Jump Host, Pageant)
  • The lower section shows the LAN Pair session timeout field instead of Unix permissions
  • The connect button label changes to Pair...

Operational notes

  • LAN Pair does not require PHP, SSH, Pageant, or any external SMB mapping
  • File operations (listing, download, upload, rename, delete, mkdir) are all supported
  • Both machines must run Total Commander with this plugin installed
