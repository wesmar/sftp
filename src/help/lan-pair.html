LAN Pair (SMB-like)

LAN Pair is a Windows-to-Windows transfer mode for direct local-network pairing without relying on the system SMB stack. It is designed for practical file exchange between trusted machines on the same network.

What LAN Pair is for

  • Quick transfer between two Windows machines on the same LAN
  • Cases where classic SMB pairing is inconvenient or blocked by local policy
  • Direct plugin-managed sessions inside Total Commander

What it is not

  • It is not classic Windows file sharing
  • It does not enumerate normal SMB shares
  • It is not routed through SSH, SFTP, PHP Agent, or PHP Shell

Roles

  • Auto - discovery/listen mode; role is chosen by the user when a peer is detected
  • Receiver - waits for an incoming paired client
  • Donor - connects to the detected receiver
Important: Auto mode does not silently decide roles for the user. When a matching peer is found, the plugin asks which role should be used.

Basic workflow

  1. Create or edit a session and set Transfer = LAN Pair (SMB-like).
  2. Set the session name. This name is also used as the visible peer label.
  3. Choose Auto, Receiver, or Donor in the role dropdown.
  4. Optional: set Session timeout in minutes. 0 means no limit.
  5. Click Pair....
  6. When the peer is found, confirm the role and continue.

Session timeout

  • 0 = no timeout
  • >0 = idle/session timeout in minutes
  • The timeout is stored per session as lanpairtimeout in sftpplug.ini
  • Timeout is checked between operations; active transfers are not force-aborted mid-stream

Saved settings

Typical LAN Pair session keys in sftpplug.ini:

[MyLAN]
transfermode=3
server=lanpair://peer/WORKSTATION-01
lanpairrole=0
lanpairpeer=WORKSTATION-01
lanpairtimeout=0

UI behavior

  • In LAN Pair mode, SSH-specific controls are disabled
  • Key-file fields and Jump Host controls are not used
  • The lower permissions block is replaced by the LAN Pair session timeout field

Operational notes

  • LAN Pair uses its own TCP/UDP discovery/session path
  • It does not require PHP, SSH, Pageant, or external SMB mapping
  • Resume support and chunked transfer behavior depend on the LAN Pair session engine, not on SFTP/SCP rules

Troubleshooting hints

  • If peers do not appear, check whether both machines are on the same network segment
  • Discovery problems are separate from normal TCP connectivity
  • If classic SMB works but LAN Pair discovery does not, investigate UDP/broadcast visibility rather than filesystem permissions
