PHP Agent (HTTP)

PHP Agent lets you manage files on a web server via HTTP — without any SSH account or SSH daemon. The entire server-side logic lives in one file: sftp.php.

Typical use case: shared hosting where you have FTP/SFTP disabled or no shell access, but PHP is available.

Important — read before you do anything:
The sftp.php included in the plugin package has no password configured. If you upload it to your server before setting a password in the plugin, the server will reject every connection with HTTP 503. Always set the password in the plugin dialog first, save the session, then upload.

How it works — the big picture

  1. You choose a password for the connection.
  2. The plugin stores a salted SHA-256 hash of that password into your local sftp.php. The password itself is never written to any file.
  3. You upload that modified sftp.php to your web server.
  4. When you connect, the plugin signs every request with HMAC-SHA256 using your password. The server verifies the signature against the stored hash. The password never travels over the network in any readable form.

This means: even if someone reads sftp.php on the server, they cannot recover your password — they only see a hash.

Step-by-step deployment

Step 1 — Set up the connection in Total Commander

Open the plugin connection dialog (Net → Connect, or the Network Neighborhood panel). Fill in:

FieldWhat to enterExample
Connect to:Full HTTPS URL to sftp.php on your server (the file does not exist yet — enter where it will be)https://example.com/sftp.php
Transfer mode:Select PHP Agent (HTTP) from the dropdown—
Password:Choose any password — this will be your shared secret between TC and the serverMySecretPass123
Session name:Give the session a name for easy identificationMy Hosting
URL format: Always use https:// in production. Plain http:// is allowed for local testing only — over HTTP all traffic is unencrypted.

Step 2 — Save the session

Click Save (or press F2 if available). At this point the plugin:

  • Encrypts the password with Windows DPAPI and stores it in sftpplug.ini
  • Opens its local copy of sftp.php and writes two fields into it: AGENT_PSK_SALT (random) and AGENT_PSK_SHA256 (SHA-256 hash of salt + your password)

The local sftp.php that now contains your password hash is located at:

...\Total Commander\plugins\wfx\SFTPplug\sftp.php

This is the file you will upload. Do not use the original sftp.php from the download package — it has empty PSK fields.

Step 3 — Upload sftp.php to your server

Copy the file from the path above to your web server so that it is accessible at the URL you entered in Step 1.

Example: if you entered https://example.com/sftp.php, upload the file to the root of your web space as sftp.php.

You can upload it using any method: FTP, cPanel File Manager, your hosting control panel, or any other file manager.

File permissions on Linux servers: Set sftp.php to mode 644 (readable by web server, not world-writable).

Step 4 — Verify the endpoint

Before connecting, verify the script is reachable and correctly configured. Open this URL in your browser:

https://example.com/sftp.php?op=PROBE

Expected responses:

ResponseMeaning
HTTP 200 with JSON containing "status":"ok"All good — proceed to connect
HTTP 401 UnauthorizedScript is running but waiting for auth — this is also OK (plugin will authenticate)
HTTP 503 Service UnavailablePSK not configured — you uploaded the wrong file (without hash). Repeat from Step 2.
HTTP 404 Not FoundWrong URL or file not uploaded yet
PHP error pagePHP version too old (<7.4) or required extension disabled on server

Step 5 — Connect

Select your saved session in Total Commander and connect. The plugin probes the endpoint, confirms capabilities, and opens the remote file panel.

What is stored where — security summary

LocationWhat is storedIs the password exposed?
sftpplug.ini (on your PC)Password encrypted with Windows DPAPI (user-account bound)No — DPAPI ciphertext only
sftp.php (on server)AGENT_PSK_SALT (random) + AGENT_PSK_SHA256 = SHA-256(salt + password)No — one-way hash only
Network traffic (HTTPS)HMAC-SHA256 signature per request + timestamp + nonceNo — signature only; replay attacks blocked
In short: Your password never appears in plain text in any file or on the network. Even if someone obtains sftp.php from the server, they cannot reverse the hash to get your password.

Capabilities

  • LIST — directory listing with name, size, modification time, permissions
  • GET — file download
  • PUT — file upload (chunked)
  • DELETE / RMDIR — remove files and directories
  • MKDIR — create directories (recursive)
  • RENAME — rename or move files
  • STAT — file/directory info
  • HASH — compute file hashes on server
  • SHELL_EXEC — execute shell commands (PHP Shell mode; requires shell_exec enabled)

PHP Shell Console

If shell_exec is available on your server, you can use PHP Shell (HTTP) transfer mode for a pseudo-terminal experience:

  • Run shell commands from within Total Commander
  • Working directory tracking across commands
  • Command history (Up/Down arrows)

Uses the same sftp.php file — no separate deployment needed.

Server requirements

  • PHP 7.4 or later (PHP 8.x recommended)
  • Standard file and directory functions (fopen, opendir, etc.) — must not be in disable_functions
  • hash and json extensions (enabled by default on most hosts)
  • For PHP Shell: shell_exec or proc_open not disabled

PHP INI limits that affect uploads

  • upload_max_filesize — maximum size per chunk
  • post_max_size — must be larger than upload_max_filesize
  • max_execution_time — increase for large files on slow servers
  • memory_limit — 128 MB or more recommended

The plugin reads these limits via PROBE and adjusts chunk size automatically.
