Security Model

Transport security

  • SSH connections (SFTP, SCP, Shell fallback, Jump Host) are always encrypted by the SSH protocol.
  • Use HTTPS for all PHP Agent / PHP Shell traffic. Avoid plain HTTP on any network you do not fully control.
  • LAN Pair uses authenticated TCP sessions (PAIR1 protocol) within the local network.

  • All decrypted password buffers are zeroed from memory (SecureZeroMemory) immediately after use.
  • Jump Host credentials are stored and encrypted separately from target-host credentials.
  • The PHP Agent PSK is stored as a salted SHA-256 hash inside sftp.php — the raw secret is not written to disk on the server.

SSH host identity verification

  • The server's MD5 fingerprint is stored in sftpplug.ini on first connection.
  • On every subsequent connection the fingerprint is compared. A mismatch raises an explicit warning dialog — you must confirm before connecting.
  • Jump Host (bastion) keeps a separate stored fingerprint from the final target host.
  • Accepting a new fingerprint should always be a deliberate decision.

LAN Pair trust model and cryptography

  • Authentication uses PBKDF2-HMAC-SHA256 with 120,000 iterations to derive a session key from the pairing password.
  • Both sides exchange HMAC-SHA256 challenge-response proofs over TCP. Neither the password nor the derived key is transmitted in plain form.
  • On first successful connection the server issues a trust token (TOFU — Trust On First Use). The plugin stores this token encrypted via Windows DPAPI, keyed per peer-pair and per Windows user account.
  • Subsequent connections use the stored DPAPI trust key — no password prompt.
  • Session timeout is enforced between operations; active transfers are never killed mid-stream.
  • All LAN Pair cryptography uses Windows BCrypt only — no third-party DLLs.

Plugin error isolation

Every exported plugin function is wrapped by an exception firewall. If an unexpected error occurs inside the plugin:

  • The exception is caught before it can reach Total Commander.
  • The current operation is aborted and an appropriate error code is returned to TC.
  • A diagnostic dialog is shown with the error type and call location.
  • Total Commander continues running normally — a plugin bug cannot crash the host process.

Key file security

  • Private key files (PEM/PPK) are read only at connection time and not stored in the session profile.
  • PPK v2 and v3 are converted to an in-memory PEM form — the conversion result is never written to disk.
  • Argon2 (PPK v3 key derivation) is statically compiled into the plugin — no separate DLL is involved.

Operational recommendations

  • Use DPAPI password storage (default) rather than plain-text for all sessions.
  • Restrict access to the TC directory containing sftpplug.ini — it holds encrypted credentials.
  • Restrict URL exposure for PHP Agent deployments; use HTTPS with a valid certificate.
  • Rotate SSH keys and PHP Agent passwords periodically.
  • Do not commit sftpplug.ini or sftp.php with live credentials to public repositories.
  • Keep the plugin and PHP runtime up to date.
Important: Security depends on TLS configuration, server hardening, and secret management together — not on the plugin alone.
