Security Model

Transport security

  • Use HTTPS for all PHP Agent/Shell traffic.
  • Avoid plain HTTP on public networks.

Credential handling

  • Client side password is stored encrypted in sftpplug.ini (DPAPI).
  • Server side sftp.php can use salted hash constants for deployment simplicity.
  • Do not commit real production secrets to public repositories.
  • Jump Host credentials are separate from target-host credentials.

SSH host identity

  • Target host fingerprint is verified and stored for SSH sessions.
  • Jump Host / ProxyJump keeps a separate stored fingerprint for the bastion host.
  • Accepting a new fingerprint should always be a deliberate decision.

LAN Pair trust model

  • LAN Pair is intended for trusted local-network scenarios.
  • Session timeout is enforced between operations, not by killing an active transfer mid-stream.

Operational recommendations

  • Restrict URL exposure where possible.
  • Rotate credentials periodically.
  • Keep PHP runtime and plugin up to date.
Important: Security depends on TLS, server hardening, and secret management together.
