Session Management

The plugin provides comprehensive session management for storing and organizing connection configurations.

Saved Sessions

Creating a Session

  1. Open connection dialog (Ctrl+F1)
  2. Enter connection settings (host, username, authentication, etc.)
  3. Press F7 or click in session field
  4. Enter session name in the dialog (Total Commander shows "New folder" - this is expected)
  5. Settings are automatically saved under that name

Editing a Session

  1. Select session from dropdown
  2. Press Alt+Enter
  3. Modify settings as needed
  4. Changes are saved automatically

Deleting a Session

  • Select session from dropdown
  • Press F8 or Delete
  • Confirm deletion

Session Storage

Storage Location

  • Sessions stored in sftpplug.ini
  • Located in Total Commander configuration directory
  • Typically: %APPDATA%\GHISLER\ or TC installation directory

Stored Information

  • Hostname and port
  • Username
  • Encrypted password (if saved)
  • Transfer mode settings
  • Proxy configuration
  • Codepage/encoding settings
  • Key file paths
  • Advanced options (compression, shell transfer, etc.)
  • LAN Pair role, peer, and timeout
  • Jump Host / ProxyJump settings

Password Security

  • Passwords encrypted with Windows DPAPI
  • Encryption is user-specific (cannot be used by other users)
  • Passwords not stored in plain text
  • Optional: Don't save password, prompt each time

Import Sessions

Click Import... to open the import menu. Sessions are imported without modifying the source data. See Import from PuTTY / WinSCP / KiTTY for full details.

Supported sources

  • PuTTY (registry) — HKCU\Software\SimonTatham\PuTTY\Sessions
  • PuTTY Portable — browse to the portable folder; plugin finds putty.reg automatically (up to 4 levels deep)
  • WinSCP (registry) — HKCU\Software\Martin Prikryl\WinSCP 2\Sessions
  • WinSCP Portable — browse to WinSCP.ini directly
  • KiTTY Portable — browse to the KiTTYPortable root; plugin finds the Sessions folder automatically (up to 4 levels deep); only SSH sessions imported

Checkbox picker

  • All found sessions are shown in a checkbox list — pick any combination
  • Select All / Deselect All buttons for bulk operations
  • Only checked sessions are imported

Path memory

  • Last 4 used folder/file paths stored in [ImportPaths] of sftpplug.ini
  • Browse dialog pre-selects the last used location automatically

Protocol Mapping

  • WinSCP SFTP: Imported as SSH (SFTP/SCP) mode
  • WinSCP SCP: Imported with scponly=1 and scpfordata=1
  • Unsupported protocols (FTP, WebDAV, S3): Skipped with summary report

Key File Compatibility

  • Imported key paths preserved as-is
  • Supports both .pem and .ppk extensions
  • PPK v2 and PPK v3 (Argon2) converted natively — no external puttygen needed
  • Passwords are not imported — you will be prompted on first connection

After Import Checklist

  1. Open imported session with Alt+Enter
  2. Verify transfer mode matches server capabilities
  3. Check proxy settings if applicable
  4. Verify key file paths are valid
  5. Test connection
  6. Adjust SCP/Shell options for restrictive hosts

Session Organization Tips

Naming Conventions

production-server1      - Clear environment prefix
dev-db-backup           - Purpose in name
clientname-project      - Client/project based
aws-eu-west-web         - Cloud provider + region

Session Groups

Use naming patterns for logical grouping:

  • Sessions appear alphabetically in dropdown
  • Prefix with category for grouping
  • Example: AWS-, Azure-, Local-

Session Limits

  • No hard limit on number of sessions
  • Practical limit: INI file size (typically fine for hundreds)
  • Each session ~500-1000 bytes in INI

Troubleshooting

Session not saved

  • Check INI file is writable
  • Verify Total Commander has write permissions
  • Check disk space

Password not remembered

  • Ensure "Save password" was selected
  • DPAPI requires same Windows user
  • Re-enter password and save again

Import fails

  • Verify source sessions exist in registry (PuTTY/WinSCP) or in the correct folder (Portable/KiTTY)
  • Check registry permissions for registry-based sources
  • For KiTTY Portable: ensure the folder contains a Sessions subdirectory with session files
  • Run Total Commander as administrator if needed
Tip: Export your sftpplug.ini periodically for backup. Sessions are portable between machines (passwords require same Windows user).
