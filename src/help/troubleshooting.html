Troubleshooting

Wrong credentials for PHP Agent

  • Verify password in connection options.
  • Re-save session to regenerate local sftp.php.
  • Upload regenerated sftp.php again.

PHP Agent endpoint not found (HTTP 404)

  • Check URL path and filename.
  • Ensure server URL points to uploaded sftp.php.

PHP Agent not configured (HTTP 503)

  • Server script has no valid auth constants/environment values.
  • Deploy freshly generated script from plugin directory.

LIST/GET/PUT failures

  • Check server open_basedir, upload limits, and file permissions.
  • Verify current working path and root restrictions.
  • Enable detailed log for diagnostics.

TAR batch download returns 0 files

  • Upload the current sftp.php from the plugin directory — old versions do not have TAR_PACK support.
  • If server still returns 0 files, verify the connection URL matches the deployed script location exactly.

HTTP 504 Gateway Timeout on TAR download (OVH / home.pl)

  • Upload the current sftp.php — the updated version streams directly without buffering and avoids server-side timeouts.

TAR upload fails for large directories

  • Update the plugin to the current build — older builds had a 4 GB limit due to a 32-bit integer overflow.

CHM does not open

  • If file comes from internet, open file properties and click Unblock.
  • Keep sftpplug.chm local, next to plugin DLL.
